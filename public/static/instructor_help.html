The FAQ is in the general help/faq for all users (not just instructors).

Typical Work Flow

  • Learn how to use the system.
    • Create an instructor account and two student test accounts. (Google ignores everything including and after a plus sign in the user name. Thus, myself+courseA@gmail.com will send emails to myself@gmail.com. You can then search or create filters to select only courseA related emails.
    • Create a few silly quizzes as instructor, fill them out as a pretend-student. In the end, just delete the results.
  • Edit a testbank until you are satisfied. Look at it via the "View Testbank" on the right side of the file manager.
  • Rename the .testbank extension to a .quiz extension.
  • Tell your students about the quiz. You can do that in class or by setting the message for your students (in your account settings).
  • Look at the results. "Run Grade Analytics" at the bottom of your file manager to obtain grades-*.csv files, which you can download and analyze.

Notes on Formatting of Questions

Questions can include standard html. Note that this means that an instructor can destroy a quiz, too, e.g., by adding a </form> tag into a question. Please don't.

Math: You can use any html code you wish. Thus, a <sup>2</sup> will produce a raised square (2). Similarly, you can use other math symbols, such as &int;, which should typeset into the integral sign, &int;, provided your students' browsers understand it. However, html is ultimately not particularly good at math. (some examples and an explanation.) The W3C working group wrote HTML Math to consider how this should be done, but I don't think it went very far. Fortunately, you can also include html <img> images that are created by latex, and they can be hosted anywhere. Images are definitely a kludge, but they are also the only render-perfect kludges I know of, even though they are not resolution independent.

Notes on the File Manager

  • Edit-ing files copies the original (previous) version into "-previous".
  • Delete-ing immediately removes all files that end in "-deleted" and "-previous". They cannot be recovered! All other files are only renamed to end in "-deleted".
